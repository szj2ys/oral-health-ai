### 1.2 技术选型理由

| 技术 | 选型 | 理由 |
|------|------|------|
| 前端框架 | Next.js 15 | SSR/SSG支持、API Routes一体化、Vercel原生支持 |
| UI框架 | Tailwind CSS | 开发效率高、样式原子化、响应式友好 |
| 组件库 | shadcn/ui | 可定制性强、无样式污染、Radix UI基础 |
| 数据库 | PostgreSQL | 关系型数据、JSON支持、Vercel Postgres |
| ORM | Prisma | TypeScript支持、类型安全、迁移管理 |
| AI服务 | Claude API | 多模态能力、医学理解强、中文支持好 |
| 存储 | Vercel Blob | 与Vercel集成、CDN加速、图片优化 |
| 缓存 | Redis | 会话存储、API缓存、消息队列 |

## 3. AI服务架构

### 3.1 方案选型

#### 方案A：第三方AI API（MVP推荐）
```
用户上传照片 → 后端转发 → Claude API / 阿里云视觉智能 → 结果 → 处理返回
```

**优点**：
- 快速上线
- 无需训练模型
- 精度有保障

**缺点**：
- 按量付费，成本较高
- 依赖第三方服务
- 定制化程度低

**成本估算**：
- Claude API: ~$0.01-0.05/次分析
- 月活1000用户 × 3次/月 = 3000次 = $30-150/月

#### 方案B：自训练模型（长期）
```
用户上传照片 → 预处理 → YOLOv8/UNet 检测 → 分类 → 结果后处理 → 返回
```

**优点**：
- 成本可控
- 可定制优化
- 不受第三方限制

**缺点**：
- 需要大量标注数据
- 训练成本高
- 维护成本高

**技术栈**：
- Python + PyTorch
- YOLOv8 目标检测
- U-Net 图像分割
- FastAPI 服务部署

### 3.3 具体实现（MVP方案）

使用Claude API进行多模态分析：

```typescript
// lib/ai.ts
import Anthropic from '@anthropic-ai/sdk';

const anthropic = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY,
});

export async function analyzeOralImage(imageBase64: string) {
  const response = await anthropic.messages.create({
    model: 'claude-3-opus-20240229',
    max_tokens: 1024,
    messages: [{
      role: 'user',
      content: [
        {
          type: 'image',
          source: {
            type: 'base64',
            media_type: 'image/jpeg',
            data: imageBase64,
          },
        },
        {
          type: 'text',
          text: `请分析这张口腔照片，识别以下问题：
          1. 龋齿（蛀牙）
          2. 牙龈红肿/出血
          3. 牙结石
          4. 牙齿排列问题
          5. 其他异常

          请以JSON格式返回：
          {
            "overallScore": 0-100,
            "issues": [
              {
                "type": "问题类型",
                "location": "位置",
                "severity": "轻微/轻度/中度/重度",
                "confidence": 0-1
              }
            ],
            "recommendations": ["建议1", "建议2"],
            "notes": "其他说明"
          }`,
        },
      ],
    }],
  });

  return JSON.parse(response.content[0].text);
}
```

### 4.2 数据库设计

```prisma
// schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// 用户表
model User {
  id        String   @id @default(cuid())
  email     String   @unique
  phone     String?  @unique
  name      String?
  avatar    String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  scans     Scan[]
  reports   Report[]
  familyMembers FamilyMember[]

  @@map("users")
}

// 家庭成员表
model FamilyMember {
  id     String @id @default(cuid())
  userId String
  name   String
  relation String  // 关系：配偶、子女、父母
  birthDate DateTime?

  user   User   @relation(fields: [userId], references: [id], onDelete: Cascade)
  scans  Scan[]

  @@map("family_members")
}

// 检测记录表
model Scan {
  id        String   @id @default(cuid())
  userId    String?
  familyMemberId String?

  // 原始图片
  imageUrl  String
  imageHash String   // 用于去重

  // 分析状态
  status    ScanStatus @default(PENDING)

  // 结果
  overallScore Int?
  issues    Json?    // 问题列表

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  user      User?    @relation(fields: [userId], references: [id])
  familyMember FamilyMember? @relation(fields: [familyMemberId], references: [id])
  report    Report?

  @@map("scans")
}

enum ScanStatus {
  PENDING      # 待处理
  UPLOADING    # 上传中
  PREPROCESSING # 预处理中
  ANALYZING    # 分析中
  COMPLETED    # 已完成
  FAILED       # 失败
}

// 报告表
model Report {
  id        String   @id @default(cuid())
  scanId    String   @unique

  // 报告内容
  summary   String   // 总结
  details   Json     // 详细分析
  recommendations Json // 建议

  // 分享
  shareToken String? @unique
  shareExpiry DateTime?

  createdAt DateTime @default(now())

  scan      Scan     @relation(fields: [scanId], references: [id], onDelete: Cascade)

  @@map("reports")
}
```
