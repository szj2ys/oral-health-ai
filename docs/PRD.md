# 产品需求文档 (PRD)

## 居家口腔健康AI初筛

---

## 1. 产品概述

### 1.1 产品名称
张二口腔AI - 居家口腔健康AI初筛

### 1.2 产品定位
面向个人用户的居家口腔健康管理平台，通过AI技术实现口腔健康的智能初筛和风险预警。

### 1.3 核心价值主张
- **便捷性**：无需预约，随时随地进行口腔健康自测
- **智能化**：AI自动分析口腔照片，识别潜在问题
- **持续性**：建立个人口腔健康档案，长期追踪健康变化
- **连接性**：发现异常时快速连接专业医疗资源

### 1.4 目标用户

#### 主要用户群体
1. **25-45岁都市白领**
   - 工作忙碌，没时间定期看牙医
   - 对健康管理有意识
   - 习惯使用数字化工具

2. **家长群体（30-50岁）**
   - 关注孩子的口腔健康
   - 需要定期检查但难以频繁去医院
   - 愿意使用科技手段辅助健康管理

3. **中老年人群（50-70岁）**
   - 口腔问题高发
   - 行动不便，居家检查更方便
   - 需要持续监控口腔健康

### 1.5 用户痛点
1. **时间成本高**：去口腔医院需要预约、排队，耗费大量时间
2. **费用昂贵**：常规口腔检查费用不菲
3. **缺乏持续性**：多数人只在牙痛时才看牙医，缺乏预防意识
4. **信息不对称**：不了解自己的口腔健康状况
5. **早期发现难**：小问题容易被忽视，发展成大问题

## 2. 竞品分析

### 2.1 直接竞品

#### 国际竞品
1. **Pearl (pearlglobal.ai)**
   - 定位：牙科AI影像分析
   - 优势：FDA认证，专业级精度
   - 劣势：面向B端（牙科诊所），个人用户无法直接使用

2. **Overjet (overjet.com)**
   - 定位：牙科AI影像平台
   - 优势：精准的病历分析，强大的数据处理能力
   - 劣势：仅面向专业牙科医生，需要专业设备

3. **DentalMonitoring**
   - 定位：远程牙科监控
   - 优势：可跟踪治疗进度
   - 劣势：需要配合正畸治疗使用

#### 国内竞品
1. **春雨医生/好大夫在线等平台的口腔科室**
   - 定位：在线问诊平台
   - 优势：用户基数大，医生资源丰富
   - 劣势：缺乏AI初筛功能，主要依赖人工问诊

2. **美团/阿里健康口腔服务**
   - 定位：O2O口腔服务平台
   - 优势：价格透明，预约方便
   - 劣势：没有AI辅助诊断

### 2.2 间接竞品

#### 智能硬件
1. **智能牙刷（飞利浦、Oral-B、小米等）**
   - 优势：日常使用，数据持续采集
   - 劣势：只能监测刷牙行为，无法直接检测口腔问题

2. **口腔镜/内窥镜**
   - 优势：可以看到口腔内部
   - 劣势：缺乏智能分析功能

| Overjet | ★★★★★ | ★☆☆☆☆ | ★☆☆☆☆ | ★★★☆☆ | 专业工具，门槛高 |

### 2.4 差异化机会

#### 必须跟进（竞品做得好的）
1. **AI识别精度**：国际竞品的模型训练更充分，识别精度更高
2. **医疗合规**：国际产品有FDA认证，合规性更强
3. **用户体验**：智能硬件的持续监测体验值得学习

#### 可以超车（竞品没做好的）
1. **C端可用性**：现有AI口腔产品都是B端工具，个人用户市场空白
2. **价格亲民**：去除中间环节，降低使用门槛
3. **本地化**：针对中国人口腔特征优化模型
4. **全周期管理**：从初筛到诊疗的完整闭环
5. **家庭账户**：支持多家庭成员管理

### 3.2 功能详情

#### 3.2.1 智能拍照引导（MVP核心）
**功能描述**：
- 实时AR引导：通过手机摄像头显示拍照框线，引导用户对准口腔区域
- 光线检测：实时检测环境光线，提示光线是否充足
- 角度校正：检测拍摄角度，提示调整
- 质量检查：拍摄后自动检查照片质量（清晰度、曝光、遮挡等）

**用户流程**：
1. 选择拍摄模式（自助/他人协助）
2. 按照引导拍摄正面、左侧面、右侧面、咬合面
3. 系统实时质量校验
4. 重拍或确认

**技术实现**：
- 前端：React + Canvas API
- CV算法：轻量级端侧模型（TensorFlow.js）
- 备选：服务端验证

#### 3.2.2 AI智能分析（MVP核心）
**功能描述**：
- 上传照片后，AI自动分析口腔健康状况
- 识别项目：
  - 龋齿（蛀牙）检测
  - 牙龈红肿/出血检测
  - 牙结石识别
  - 牙齿排列评估
  - 牙齿色素沉积
  - 缺牙/残根检测

**输出结果**：
- 综合健康评分（0-100分）
- 各项指标的详细评分
- 问题区域标记（在照片上标注）
- 严重程度分级（轻微/轻度/中度/重度）

**技术方案**：
- 基础版：集成第三方AI API（阿里云/腾讯云/Claude API）
- 进阶版：自训练YOLOv8目标检测模型
- 高端版：多模态大模型（GPT-4V/Claude 3）

#### 3.2.3 健康报告生成（MVP核心）
**功能描述**：
- 生成可视化健康报告
- 包含：
  - 健康评分总览
  - 问题详情说明
  - 个性化护理建议
  - 就医建议

**报告模板**：
```
┌────────────────────────────────────┐
│  张二的口腔健康报告                  │
│  检测日期：2024-03-11               │
├────────────────────────────────────┤
│                                    │
│  综合评分：78/100                   │
│  [可视化进度条]                      │
│                                    │
├────────────────────────────────────┤
│  发现的问题：                        │
│  1. 牙龈红肿（轻微）- 左下磨牙区      │
│  2. 牙菌斑（轻度）- 门牙内侧          │
│                                    │
├────────────────────────────────────┤
│  护理建议：                          │
│  1. 使用软毛牙刷，注意牙龈清洁         │
│  2. 每天使用牙线                      │
│  3. 建议3个月内预约牙医检查            │
│                                    │
└────────────────────────────────────┘
```

#### 3.2.4 健康档案管理（Phase 2）

## 4. 用户体验设计

### 4.1 设计原则
1. **简洁第一**：降低使用门槛，老年用户也能轻松上手
2. **专业可信**：医疗类产品需要建立信任感
3. **温暖关怀**：避免冰冷的机器感，给予用户关怀
4. **隐私优先**：健康数据敏感性高，处处体现隐私保护

### 4.2 视觉风格
- **主色调**：蓝色系（医疗、信任）+ 白色（干净、专业）
- **辅助色**：绿色（健康）、橙色（提醒）、红色（严重）
- **圆角设计**：友好、亲和
- **插画风格**：温暖的医疗插画

### 4.3 关键页面

#### 4.3.1 首页
- 品牌展示
- 核心功能入口
- 使用流程说明
- 信任背书（医疗合规、数据安全）

#### 4.3.2 检测页面
- 步骤引导（1.拍照 2.分析 3.报告）
- 拍照界面（全屏相机）
- 分析等待页面（动画+进度）
- 结果展示页面

#### 4.3.3 报告页面
- 评分卡片（突出显示）
- 问题列表（可展开详情）
- 建议区域
- 分享/下载按钮

#### 4.3.4 历史页面
- 时间轴列表
- 趋势图表
- 筛选功能

## 6. 商业模式

### 6.1 定价策略

#### 免费版
- 基础AI分析（每月3次）
- 基础报告
- 30天历史记录

#### 会员版（¥9.9/月 或 ¥99/年）
- 无限次AI分析
- 详细报告+PDF下载
- 无限历史记录
- 家庭账户（最多5人）
- 专属客服

#### 单次付费（¥2.99/次）
- 适合低频用户
- 完整报告

### 6.2 其他变现
1. **医生导流分成**：推荐用户到合作医院，收取导流费用

4. **企业服务**：为企业员工提供口腔健康管理服务

## 7. 合规与风险

### 7.1 医疗合规
- 明确标注"本工具仅供参考，不能替代专业医生诊断"
- 不给出具体治疗方案
- 严重问题必须建议就医
- 考虑申请二类医疗器械资质

### 7.2 数据合规
- 用户健康数据加密存储
- 明确隐私政策
- 用户授权机制
- 数据删除功能
- 符合《个人信息保护法》《数据安全法》

### 7.3 风险提示
- 误诊风险：AI可能存在误判，必须告知用户
- 延误治疗风险：用户可能因初筛结果良好而忽视严重问题
- 隐私泄露风险：口腔照片属于生物识别信息
